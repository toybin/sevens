**Single table**: `triples(subject, predicate, object)` with 3 indexes. No other tables. 310+ triples serving 24 nodes including tree structure, content, cross-references, context files, computed metrics, lineage, and suspension state.

**Morphism primitives**: `Compose` (forward-forward), `ComposeInverse` (forward-inverse) in SQL. Path specs with `~` suffix for inverse traversal.

**Engine**: `mo.Either[Suspension, StepResult]` as universal return type. Composed functions delegate via `:fn`. Map-over via `:map-over` predicate. Suspensions serialized as triples, resumed from DB state.

## What's Built

### Storage
- Triples table as sole store (old nodes/cross_refs/roots tables removed)
- Triple primitives: insert, query, compose, inverse compose
- Content-derived triples computed on sync (char-count, token-estimate, heading-count, wiki-link-count, has-questions)
- Lineage events as triple subjects (replaced EDN log files)
- Suspension state as triples (pipeline pause/resume survives process restart)
- `sevens query <sql>` for ad-hoc queries with table-formatted output
- `sevens search <query>` for substring search across titles and content

### Execution Engine
- `mo.Either[Suspension, StepResult]` вЂ” left = paused at gate, right = completed
- Composed functions: `:fn "decompose"` delegates to another function's full pipeline
- Map-over: `:map-over "node/parent~"` applies a function to each child
- Suspension serialization: pipeline state persisted as triples, resumed on `sevens accept`
- `deepen` function: composed `decompose` в†’ `map elaborate` over children

### Context Resolution
- Path specs: `{:path ["node/parent" "node/parent~"] :exclude-self true :with ["node/content"] :as "siblings"}`
- `~` suffix for inverse morphism traversal
- Path evaluator composes morphisms left-to-right over the triples table
- `:requires` (role-based) and `:context` (path-based) coexist вЂ” path specs preferred
- Auto-include parent, siblings, children content without manual `--include`

### Features
- Focus sessions with `.` shorthand, `--include`/`--exclude`
- `--include` on apply for one-shot context
- `--model` flag for per-call model selection (named profiles or raw model ID)
- `--yes`/`-y` to skip cost confirmation
- Cost threshold auto-approve (default $0.01)
- Streaming text output to terminal
- Discussion continuation with `[agent]`/`[user]` annotations
- Fuzzy edit matching with confirmation fallback
- `sevens revert <node>` вЂ” undo via git
- System prompt per output type (no more JSON bias for text functions)
- Validation hints for overflow and length violations
- Global + per-node context files
